## What I understood

- An index is a structure that speeds up data retrieval using mapping.
- The DB decides if using indexes would be more efficient than a full scan calculating cardinality.
- Low cardinality means indexes are usually not useful.
- High cardinality means indexes are very useful.
- Indexes work best when the query returns few rows and there are many distinct values in the rows.
- For small range queries indexes are used.
- For large range queries a full scan is more effective.
- A composite index is ordered by multiple columns.
- Leftmost prefix rule: A composite index can only be used starting from the leftmost column.
- Applying functions to indexed columns breaks index usage.
- Function-based indexes can fix this if the function is deterministic.
- Indexes involve slower inserts/updates

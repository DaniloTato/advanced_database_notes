## What I understood
- A transaction is a group of operations that must be executed as a single unit (all succeed or all fail).
- COMMIT permanently saves changes to the database.
- ROLLBACK undoes all changes made during the transaction.
- SAVEPOINT allows partial rollback to a specific point instead of undoing everything.
- Transactions ensure atomicity, meaning partial changes should never remain in the database.
- Procedures are typically used to perform transactional operations (like updates), while functions are used to return values and should avoid side effects.

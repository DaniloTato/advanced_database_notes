## What I understood
- Oracle stores metadata about all schema objects in system views such as USER_OBJECTS, USER_TABLES, and USER_DEPENDENCIES.
- DBMS_METADATA.GET_DDL can recreate the SQL definition of tables, indexes, sequences, procedures, triggers, and other objects.
- Database migrations require objects to be recreated in a specific order because of dependencies between tables.
- Even without DBA privileges or Data Pump access, it is still possible to perform schema backups using only SQL.
